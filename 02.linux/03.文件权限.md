# 文件权限

1. 查看当前用户

   ```shell
   # 当前登录的用户
   [root@zhuRoot ~]# whoami
   root
   ```

2. 文件权限修改

   ```shell
   1. 文字设定法
   chmod who[+|-] [mode] 文件名
   # 解释
   chmod:授权命令
   who: 
       文件所有者 u
       文件所属组 g
       其他人 o
       所有人 a
   对权限怎么操作：
       + 添加权限
       - 减少权限
       = 覆盖原来的权限
   mode:
       r 读
       w 写
       x 执行
      
   [root@zhuRoot ~]# ll
   --w-------. 1 root root 12744 5月   9 13:31 a.out
   [root@zhuRoot ~]# chmod a=w a.out 
   [root@zhuRoot ~]# ll
   --w--w--w-. 1 root root 12744 5月   9 13:31 a.out
   [root@zhuRoot ~]# chmod a=r a.out 
   [root@zhuRoot ~]# ll
   -r--r--r--. 1 root root 12744 5月   9 13:31 a.out
   
   
   2. 数字设定方法
   [root@zhuRoot ~]# ll
   -rwx--x--x. 1 root root 12744 5月   9 13:31 a.out
   
   [root@zhuRoot ~]# chmod 700 a.out 
   [root@zhuRoot ~]# ll
   -rwx------. 1 root root 12744 5月   9 13:31 a.out
   
   
   # 使用减法，对应的授权减去0 减去0 减去1 得到的值
   比如授权为777-001=776
   比如授权为777-101=676
   
   [root@zhuRoot ~]# chmod 777 a.out 
   [root@zhuRoot ~]# ll
   -rwxrwxrwx. 1 root root 12744 5月   9 13:31 a.out
   
   [root@zhuRoot ~]# chmod -111 a.out 
   [root@zhuRoot ~]# ll
   -rw-rw-rw-. 1 root root 12744 5月   9 13:31 a.out
   ```

3. 修改文件的所属组和所有者

   ```shell
   # 只有文件所有者和root 用户才可以更改权限
   chown 用户名 文件名
   chown 用户名:用户组  文件名
   创建用户
   useradd 用户名
   ```

4. 目录必须有执行权限，其他用户才能进去操作，否则无法操作

5. 创建软连接

   ```sehll
   ln -s /home/a.txt a.test
   注意事项：
   1. 软连接需要使用绝对路径
   2. 软连接的大小源文件+路径 的总字节数
   3. 目录可以创建软链接，硬链接不可以
   
   ```

6. 软硬链接的区别

   ```shell
   软连接占用的空间时存储路径的空间，
   硬链接不占用空间。我们看到的ls -lh 看到的信息发现占领空间了，这是因为ls -lh这个命令的问题，统计文件大小不从磁盘统计，而是根据文件属性中的大小叠加得来的。
   使用du -h 可以进行查看磁盘有关的统计信息
   ```
